# CI Performance Tracking And Visualization

Status: implemented CI benchmark tracking for PR baseline comparison and
main-branch trend visualization. This design tracks end-to-end runtime and the
main component timings of the workflow.

## Goals

- Keep performance as a first-class design constraint for trajectory simulation
  and hazard-map generation workflows.
- Make PR performance regressions visible relative to the latest main baseline.
- Publish a directly embeddable main-branch trend chart for README visibility.
- Track component timings, not only one aggregate runtime.

## Tracked Components

The CI aggregation extracts these benchmark components from
`scripts/run_performance_benchmark.py` summary output:

- total workflow wall time (all stages),
- terrain loading,
- release generation,
- simulation kernel runtime,
- validation output writing,
- hazard-stage total runtime,
- hazard accumulation,
- hazard output writing,
- hazard bounds discovery.

## CI Workflows

### PR Workflow

Workflow: `.github/workflows/performance_pr.yml`

1. Runs the standard synthetic benchmark profile.
2. Aggregates component timings.
3. Fetches the latest main baseline from GitHub Pages
   (`/performance/latest.json`).
4. Writes a PR comparison report with per-component deltas.
5. Publishes the table into the workflow summary and stores JSON/Markdown
   artifacts.

Positive delta means slower runtime; negative delta means faster runtime.

### Main Workflow

Workflow: `.github/workflows/performance_main.yml`

1. Runs the same standard synthetic benchmark profile on each push to `main`.
2. Loads existing published history from GitHub Pages
   (`/performance/history.json`).
3. Appends or refreshes the current commit entry.
4. Regenerates:
   - `history.json`,
   - `latest.json`,
   - `main_performance.svg`,
   - `index.html`.
5. Deploys these files to GitHub Pages.

## Rendering

The chart is generated by `scripts/performance_ci_tracking.py` as a lightweight
SVG line chart with multi-series component overlays:

- Total,
- Simulation,
- Hazard accumulation,
- Validation write,
- Hazard write.

The README embeds `main_performance.svg` directly from Pages.

## Operational Notes

- Benchmark profile is intentionally fixed to `standard` for stable, short CI
  execution.
- The history list is capped to avoid unbounded growth.
- These timings are CI diagnostics; they are not scientific validation metrics
  and should be interpreted across comparable runners.
